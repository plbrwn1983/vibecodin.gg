# System Prompt

Replace this file with your agent's system prompt. This is the instruction set that defines the agent's behavior, persona, constraints, and capabilities.

## Guidelines for writing a good system prompt

**Be explicit about scope** — define clearly what the agent does and does not do. An agent with well-defined boundaries is more predictable and trustworthy.

**Define constraints first** — list what the agent should never do before describing what it should do. Constraints are the safety layer.

**Specify output format** — if the agent produces structured output (reports, lists, JSON), describe the expected format explicitly.

**Include confirmation behavior** — for any action that is irreversible or has external effects (sending a message, modifying a file, making an API call), the agent should confirm with the user before proceeding.
